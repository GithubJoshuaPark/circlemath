# 2026-07-05 정수와 유리수 노트북 LaTeX 렌더링 에러 수정 기록

이 문서는 주피터 노트북 파일([01_02_integers_rational_numbers.ipynb](file:///Users/joshuapark/Desktop/Dev/soromiso/circlemath/middle_high_school_math/01_middle_1/02_integers_rational_numbers/01_02_integers_rational_numbers.ipynb)) 실행 중 발생한 Matplotlib 라벨의 LaTeX 수식 파싱 에러(ParseException)를 조치한 내역을 기록한 문서입니다.

## 1. 에러 현상 및 원인
- **에러 로그**: `ValueError: 무리수 $1/\sqrt{2}$ ($pprox 0.7071$)` 및 `ParseException: Expected end of text, found '$'`
- **원인 분석**:
  - Python f-string 형식의 문자열 리터럴을 JSON에 쓰고 다시 컴파일하는 과정에서, LaTeX의 백슬래시 기호 `\approx` 및 `\sqrt{2}`가 파이썬 이스케이프 제어 문자와 충돌을 일으켰습니다.
  - 구체적으로 `\approx` 내부의 `\a`가 아스키 벨(Bell, \x07) 제어 문자로 컴파일되면서 문자열이 깨져 `$pprox 0.7071$` 형태로 인식되었고, Matplotlib의 수식 파서가 이를 올바른 LaTeX 수식 문법으로 해석하지 못해 에러가 발생했습니다.

## 2. 해결 방안 및 조치
- `.ipynb` 형식 파일은 직접 수정할 수 없으므로 `notebooks` MCP 서버의 `replace_cell` 도구를 사용하여 Cell 2(유리수 조밀성 시각화 코드)의 코드를 수정하였습니다.
- 수식 문자를 안전하게 렌더링하기 위해 f-string과 이스케이프 기호의 혼용을 차단하고, 백슬래시가 그대로 보존되는 **Raw String (`r'...'`)**을 사용하여 문자열을 명시적으로 결합하도록 수정하였습니다.
  - **기존**: `label=f'무리수 $1/\\sqrt{{2}}$ ($\\approx {irrational_val:.4f}$)'`
  - **수정**: `label=r'무리수 $1/\sqrt{2}$ ($\approx$ ' + f'{irrational_val:.4f})'`
- 수정 과정에서 복사-붙여넣기 실수로 Y축 변수가 `raster_sizes := sizes`로 잘못 바뀐 사소한 오타도 발견하여 `rationals_y`로 완벽히 원복 시켰습니다.
